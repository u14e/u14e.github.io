写在2016年的最后一晚上

发表于 2016-12-31   |  

前言

16年马上就要结束了，总觉得该写点什么。思前想后，自己这一年好像没做多少东西，所以这篇文章不能算是一篇总结，顶多算是一个记录。
所以本文主要对自己这一年做个记录，然后对即将到来的17年，做个大致的规划。

16年回想

从捧起《Head First HTML与CSS》到现在，算起来也差不多一年了

还记得去年的此时，自己还是个背着沉重的电脑每天出入某大学图书馆的无业游民。从最简单的html与css开始边看书边跟着敲代码，大概四个月后，算是学完了前端三大基础——html\css\javascript，以及jquery，也尝试仿写了一个静态网站。正好赶在春招旺季的尾端开始投简历找工作，皇天不负有心人，终于在几天的一天两面的节奏下，顺利拿到一个比较满意的offer，一直觉得自己是个很容易满足的人，在拿到还算满意的offer后，就再也不想投简历、参加后面的面试了。终于在四月中旬，顺利进入人生第一个职场，真的很庆幸自己有机会能够进入这个公司，虽然和脑中意淫的公司有很大差别(毕竟是一个还没入行的门外汉的瞎想)，但是在这家公司真的学到了很多东西。从刚进入公司，同事帮忙搭建开发环境，看到偌大一个项目的茫然，一天一个新概念，svn、nginx、bootstrap、coffeescript…到开始修改bug，从最简单的错别字修改（没错，就是错别字，因为我当时连页面的相关代码在哪个文件都找不到），刚入职的一个月里，每次下班回家内心都特别虚，感觉自己实在太菜了会不会被裁掉，还好有后摇帮助舒缓情绪。然后正式工作就是从调试bug开始，在long哥的指导下学会了使用chrome打断点，定位问题，调试bug。再后来又学习了react+redux+es6+webpack，并且在long哥的指导下和后端合作，完成了两个算是练手的还不算项目的小项目，之后项目代码迁徙到git上，又学了点git，也参与了公司产品的一个迭代版本的前端开发，那时候真是在svn和git，以及node 0.12.4和node 5.1.0两头切换的混乱状态。至此，终于算是到了一个初级前端工程师的水平(自认为的)

10月因为一些不可抗拒因素，注定要结束这短暂的“学习”，因为有一个月的交接期(其实啥事都没有)，外企的好处就是流程很规范，个人时间很充足。所以开始考虑着下面该去什么样的公司，以及自己应该学会哪些技术，在技术老大、long哥的指点，以及自己的思考下，从十一开始去图书馆借了几本node书，决定学学node，了解下服务端开发。终于在中旬用node的express，简单的mongodb加react那一套写了个具备简单的文章的CRUD，以及简单的node爬虫数据展示功能的网站，以为自己学得不错，找工作应该很轻松了，然而还是too young too simple（long哥说这东西太简单了，没多大意义），发出去的简历，一周都没几个面试机会，大部分公司直接标为不合格。还好在十月底终于找到了工作，也是两个offer中比较满意的一个offer

11月入职新公司，环境感人，有种在小作坊的感觉。不过有一点好的是，同事都很年轻，而公司作息和外企也差不多，还算比较自由。从入职到现在，又接触了weui+antd两套ui组件，参与了一个基于微信公众号的微型商城+商城后台的前端开发，算是写了不少代码，每天都在实践中成长，也适应了这种开发模式，期间也开始了组件的封装，这算是踏上了又一个向上的台阶了。

11月底，由于对pyhon的魂萦梦牵(之前看过一本入门书)，加之找到了python官方教程的中文版，所以就趁着空闲把教程啃完了，觉得不过瘾又去京东买了一本比较厚的python入门书，语法部分又看了一遍，然而，后期因为一些个人情感原因，一直没有学习的欲望，所以python热就这样搁置了。

然后现在每天的生活大抵就是在公司写代码到下班，然后回来八点多，看两集《银魂》，发会儿呆，就洗洗睡了。相当颓废。

总的来说，这一年对自己来说，学了不少东西，但是就如long哥所说，你要专注一个领域，太多东西只会皮毛没有太大意义。所以这一年算是到处招花惹草，明年就该老老实实专注一个领域，做深入研究和积累了。

最后一句：16结束了，css还是写的跟屎一样，还是有点抗拒写css，js和react还只是初级阶段，还需要深入

17遐想

明天就是17年了，自己的生活状态经过一个多月的调整，也算是进入了正轨。想了想，17年有太多的事要做，不过主题只有一个——专注与沉淀。下面从几个方面说下自己接下来的打算

技术博客

技术博客既包括别人的博客文章，也包括自己的博客。其实自己在工作前就有博客，主要是学习的记录。但是工作之后，就没怎么更新了。一方面是懒，一方面是没有太多时间学习体系化的知识以及积淀，当然都是借口，主要还是因为不想走出舒适区。所以接下来一年应该会把积淀下来的知识通过博客记录下来，当然不限技术，也会心血来潮写些内心独白。至于别人的博客，之前其实一直有关注博客，阅读博客的习惯的，还专门用FeedDemon订阅每天的博客更新，也许是信息依旧爆炸，或者还是因为比较懒，最后也不了了之了。鉴于还是不习惯电脑阅读，所以今天去亚马逊买了个kindle，没错就是今天。计划着，以后通过玩手机、逛社区的间隙收集一些有用文章，统一推送到kindle，然后kindle读完之后，感觉有收获的，会通过个人公众号(公众号早就开通后，只是一直躺着)推送出来，算是起到一个收藏的功能吧！

不知道这个流程，自己的频率可以控制在一周几次。不过，先尝试着做着看看吧

领域的选择

一年前以为前端主要用css写漂亮的界面，但是工作之后，发现自己完全错了，不过也很庆幸，自己90%的时间都在写js。昨天读到一篇2016前端开发技术巡礼，里面介绍了几种可能的侧重点和技术要点，比较好理解的是大数据和WebVR(其他的水平有限不太理解)，鉴于自己大部分工作也是和前端可视化有关，而且比较感兴趣，以及对图形，炫酷交互的抵触，果断还是选择数据可视化

技术点

17年可能不会再关注其他语言了，重心都会放在js以及相关的框架react生态上面，毕竟16年也算是接触了python(虽然只是入门)。

下面简单列举几点：

  • es2015+es2016+es2017（据说17年又有一大波新特性）
  • css3技巧
  • react+redux+webpack
  • 函数式编程
  • nodejs+koa2
  • http协议
  • javascript设计模式
  • canvas+WebGL

学习方式

上面列举的几点自认为还算是比较实用的，至于学习方式这里也列举几点，算是给自己定个目标了

  • 阮一峰《es6标准入门》(第三版)(据说17年出版覆盖最新特性)
  • 《你不知道的javascript》系列
  • react+redux+webpack官方文档 + 知乎专栏 + segmentfault专栏
  • antd组件源码、semantic/bootstrap4源码
  • nodejs官方文档
  • github demo
  • 《图解http》
  • 《javascript设计模式与开发实践》

上面列举的几点仔细想想也算是废话，不过对于自己废话也好过没话。其实就是js语言多看书，react框架熟读文档，阅读别人的项目以及源码，当然英文的东西也得下定决心去啃了

总结

说了这么多，不知道自己能不能在明年的此刻完成现在的设想。也许想法还是太天真，也许前方困难重重，也许还会有些天灾人祸，但是既然自己有了想法，就试着去做一下，结果或许很重要，但是探索的过程同样也会有意想不到的收获。
