使用hexo+github建立自己的博客

发表于 2016-10-21   |  

前言

其实刚学编程时，就尝试过按照网上的教学手把手傻瓜式的折腾一个hexo+github pages出来。结果显然是因为什么都不懂，遇到问题也不知道该从哪里开始着手，所以每次重复操作都以失败告终。现在工作有段时间了，算是对编程有些领悟了，刚好有时间，就重新尝试了下，结果还真成功了，而且还很简单。所以这里记录下我搭建的整个过程，不是教别人，只是记录。如果被别人看到，并有所借鉴，那是极好的，毕竟我也是借鉴别人的经验才做成的。

域名购买

  1. 首先是去Godaddy搜索自己想好的域名（比如我的是u14e.me）
  2. 如果还没被出售就选择，然后进入购物车
  3. 其他的什么保护隐私、建站达人、虚拟主机都不用选，然后继续进入购物车
  4. 期限看自己情况，我选的是一年
  5. 看到价格还是那个价格就说明你没有买其他的附加产品，优惠码网上搜索一下，一般都会优惠一点
  6. 然后结算，新页面中选择新用户，在结算页面填好信息，继续就可以了…

DNS解析

  1. 去DNSpod注册账户
  2. 选择左侧栏的域名解析
  3. 进入控制台，选择添加域名，键入你的域名
  4. 添加两条记录(记录值指向的IP地址是github pages的IP)

修改域名对应的DNS地址

  1. 去Godaddy，右上角个人账号的下拉菜单里面选择“我的产品”
  2. 然后在域名那一项选择“管理”
  3. 新页面的域名区域右上角下拉菜单选择“管理DNS”
  4. 域名管理器下修改为自定义域名服务器，添加

目前为止完成了域名购买与解析以及与github绑定

大概的一个思路就是：现在DNSpod将你的域名和github pages的IP绑定，然后把Godaddy上面的域名管理器，换成DNSpod。这样用户键入网址时，浏览器会通过DNSpod解析到IP，然后通过IP查找到github pages服务器，从服务器上获取到数据。

hexo安装和生成

首先需要知道hexo的常用命令

  • hexo g === hexo generate (生成资源，就是网页到时候请求的数据，在public文件夹下)
  • hexo clean (清楚生成的资源)
  • hexo s === hexo server (生成资源后，开启本地服务，可以在http://localhost:4000/预览效果)
  • hexo d === hexo deploy(生成资源后，部署到服务器)
  • hexo n === hexo new (新建页面，默认在/source/_post/下面生成文件)
  • hexo new page "pageName"（新建页面，同上）

常用命令组合

  • hexo d -g (生成并部署)
  • hexo s -g （生成并本地预览）

步骤：

  • 安装nodejs、git
  • 安装hexo（可参考官网hexo）(在新建文件夹下执行下列命令)
  • 生成
    hexo g
  • 本地预览
    hexo s (现在就可以去浏览器输入http://localhost:4000/看到效果)

  • 复制主题（以next为例）

  • 修改主题(详细配置可参考next官网)

  • 写文章

配置和使用github

这里略过github的注册，并假设你已经创建了诸如u14e.github.io这样的仓库

要想从本地push代码到github仓库，必须将本机的ssh key添加到github上去

  • 首先查看本地有没有id_rsa.pub

  • 如果没有就创建一个，然后一路回车，中途要输入密码，（密码是在提交项目时使用的，如果为空的话，提交项目就不需要输入，设置密码是防止别人往你的项目里提交内容）

  • 复制id_rsa.pub文件内容

  • 添加到github上，在settings -> SSH and GPG keys里面new ssh key，将复制的内容添加进去就可以了

  • 测试

  • 设置用户信息

部署到u14e.github.io上

  • 在根目录的_config.yml里面添加git仓库（提前安装npm install hexo-deployer-git --save）

  • 执行hexo g生成资源，然后hexo d部署到github上，浏览器输入u14e.github.io就可以看到了

  • 但是现在你的域名u14e.me还是不能访问，需要添加一个文件CANME，放在source文件夹的根目录下，方便生成资源时生成到public文件夹的根目录下

  • CANME文件里面加上你的域名u14e.me，然后部署

  • 现在当你再次键入u14e.github.io时，浏览器会自动跳转到u14e.me页面
